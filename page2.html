RT Request Tracker

Faisal, our mock client, requested a ticketing system be implemented in order to be used over the next semester. A replacement was needed for the current 'To Do List' system being used through Teams, largely due to how regularly it was ignored. The RT Request Tracker ticketing system exists as a student managed program which will be maintained ongoingly, in support of both Project as well as the Operations Engineering class.

The finished product is a website capable of handling tasks for multiple different groups subsequently, relying on a virtual machine running Ubuntu 20.04 to host. Accounts for each user in a class will be created, assigning them then into individual groups. These groups will act as classrooms, acting separately from one another. Users are able to create tickets, assign them to other users or mark them as resolved. This is backed by an email alert system constantly keeping the group up to date.

I begun working on the task with fellow student Roy, however his priorities swapped midway through the semester onto obtaining a virtualization certificate. As a result of this the majority of the work done on the project was solo. While there is minor work and maintenance required in the future, I have ensured documentation and instruction is fully supplied and available in order to make the transfer as smooth as possible.

Difficulties encountered

  • What will we use?
    A research document was written up, discussing options and giving a recommendation. After this, deployment options were investigated.
  • Documentation was not available
    Trial and error as well as digging through the configuration guides. Eventually wrote my own tutorial for and use.
  • Dependencies were not working
    VM was recreated using Ubuntu 20.04 as well as installing dependencies via manual input over rather than the supplied script
  • Cross Site Request Forgery
    IP addresses and registered DNS were whitelisted for access within the RT5 config files.
  • Domain Name System
    Gained access to op-bit.nz Azure DNS service and registered under Isaacrt.op-bit.nz with a fixed attached IP address.
  • Backups
    Backups were done using Azure's storage service. Proposal created for backup implementations next semester.
  • Email Service
    Routed RT5's alert system through a PostFix email service, configured through a series of RT5 scrips.
  • SSL
    Gained SSL files from Teams to our relevant DNS. Manually rewrote RT5 config files to enable site security.
